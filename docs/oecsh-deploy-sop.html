OEC.SH — Standard Operating Procedure

Enable Odoo Enterprise → connect a GitHub repo → deploy a custom module → install in Odoo. For the laundryx.app Odoo 19 project on OEC.SH.

OEC.SH is a SEPARATE hosting from Cloudpepper with a different process — do not mix the two. For the Cloudpepper procedure see cloudpepper-free-tier-sop.html. For what the module does / how to use it, see laundry-pos-module-guide.html.

The <UUID> is not stable. OEC.SH generates a new stack UUID every time the environment is deleted/recreated, and the database name equals the UUID. Never hardcode it — discover the current one first (run from PowerShell):

2. Enable Odoo Enterprise

Beyond the official enable-enterprise guide, two steps matter:

  1. Org Settings → Git Connections: add a GitHub Personal Access Token (classic) with scopes repo + read:org. This is the "GitHub — Enterprise access" connection used to reach the private Enterprise repo. (A plain OAuth connection alone was not enough for Enterprise.)
  2. Org Settings → Addon Repos for the Enterprise repo (https://github.com/odoo/enterprise.git, branch 19.0): you MUST tick "Install dependencies from this addon" (it reads requirements.txt/apt.txt).
Undocumented gotcha: the official guide does not mention the "Install dependencies from this addon" checkbox. Skipping it the first time caused the database to fail to load and not register as Enterprise. Always enable it.

3. Connect your GitHub repo (Your ERP Code)

  1. Org Settings → Git Connections: connect GitHub (OAuth is fine for your own repos).
  2. Project → Repositories → "Your ERP Code" (the Main Repository): click Connect, pick the GitHub connection, select repo pinoylad1985/laundry-pos-odoo, branch main, Save.
"Your ERP Code" is "Always included" in every environment automatically. Do NOT also add it under the environment's Deployed Repositories / Add Repository — that section is only for additional repos.

4. Deploy to the environment

This is the PRODUCTION environment (live laundromat). Deploying clones the repo and restarts Odoo — do it during a quiet moment, not mid-transaction at the POS.
  1. Environment (production) → Repos → Environment Repositories: click Deploy.
  2. Watch the Logs tab. Success = the addons paths line includes /mnt/extra-addons/laundry-pos-odoo and Odoo restarts cleanly.
Auto-deploy toggle (Main Repository): leave it OFF on production so you control timing. Turning it on means every push to main goes live automatically.
The addons path points to the repo root (/mnt/extra-addons/laundry-pos-odoo); the module folder laundry_pos/ sits inside it, so Odoo scans the root and detects it. Repo-root layout is correct.

5. Install the module in Odoo

Deploying makes the files available but does not auto-install. Follow the host-agnostic Module Guide: Developer Mode → Apps → Update Apps List → search Laundry POS → Install.

6. Why the database name is a UUID (not laundryx)

OEC.SH auto-generates the database name from the environment <UUID>, set as db_name/db_host in odoo.conf with a matching dbfilter. This is by design — it supports multi-environment cloning (each clone needs a unique DB). laundryx is just the project/subdomain label; the Postgres DB underneath is the UUID. (Contrast Cloudpepper, where the DB is literally named laundryx.) You did nothing wrong. Note the UUID changes whenever the environment is recreated — see the warning in section 1.

7. Admin password recovery

Admin login on this DB: user id 2 = pinoylad1985@gmail.com. Discover the current <UUID> (section 1), then run from PowerShell on your PC — it resets user id 2 and prints its login:

ssh root@152.42.176.183 "docker exec -i <UUID>_odoo odoo shell -d <UUID> --no-http <<'EOF'
env['res.users'].browse(2).write({'password': 'YOUR_NEW_PASSWORD'})
env.cr.commit()
print('Password reset for login: ' + env['res.users'].browse(2).login)
EOF"
You must pass --no-http, otherwise the shell tries to bind port 8069 (already used by the running Odoo) and crashes with Address already in use.
Targets user id 2 (the admin) regardless of its login name, so it works whether the DB is a fresh Odoo (admin) or your restored backup (pinoylad1985@gmail.com). The printed login tells you which username to use.
