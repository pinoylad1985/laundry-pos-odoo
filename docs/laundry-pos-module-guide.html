1. What the module does

laundry_pos turns the standard Odoo POS into a laundry intake terminal:

  • A New Order modal that opens automatically for every new order and walks the cashier through customer, services, service type and schedule.
  • An auto-calculated turnaround (Express / Regular) based on the schedule.
  • Laundry-aware cart rules (per-KG Wash-Dry-Fold, single-item Dry/Wet & Shoe Clean, configurable options).
  • A multi-copy laundry receipt (transaction / shop / customer copies).
  • Cash accountability controls when opening and closing the register.
Note for anyone who used an earlier version: there is no longer a "Laundry Services" tab on products and no per-service product filtering. The module recognises four fixed service products by name — see below.

2. Installing the module in Odoo (~2 minutes)

The code must already be deployed to the server's addons path — see the host-specific deployment guide first.
  1. Open your Odoo instance
  2. Settings → scroll to the bottom → Activate developer mode
  3. Apps → click Update Apps List (confirm the dialog)
  4. In the Apps search bar, remove the default "Apps" filter chip (it hides non-application modules)
  5. Search for Laundry POS → Install

3. The 4 service products

Create these as normal POS products (the module finds them by name, so the names must contain these words):

Product name containsBehavior
Wash-Dry-FoldCharged per KG (editable weight; minimum applies at payment)
Dry/Wet CleanOne item per line (quantity locked to 1); long turnaround
Shoe CleanOne item per line (quantity locked to 1); long turnaround
PressQuantity may be more than 1

Add product attributes/variants (e.g. a "Turnaround" attribute) as needed — the cashier picks options in the cart, and the turnaround value is set automatically from the schedule.

4. Taking an order (the New Order modal)

When the cashier starts a New Order, the modal opens automatically with four steps:

  1. Customer — choose New or Returning. For returning, search by name / phone / address and pick the customer (or edit their details).
  2. Services — tap the service pills (Wash-Dry-Fold, Dry/Wet Clean, Shoe Clean, Press). Each tap adds that product to the order. Tap more than once to add multiple lines.
  3. Service Type — Drop-off / Drop-off & Delivery / Pickup & Delivery / Locker / Self-service.
  4. Schedule — pick the date (Today / Tomorrow / +2 days) and hour. Drop-off uses a Claim time; the others use Pickup and/or Delivery times. Some early-morning hours are closed.

The modal then shows the calculated turnaround — ⚡ Express or 🕐 Regular — and blocks confirmation if the schedule is below the minimum lead time.

  • Continue — saves the order details; they appear in the on-screen banner and on the receipt.
  • Skip for now — keeps what was entered but leaves setup incomplete. Until setup is completed, adding products and choosing a customer are blocked (the banner flashes).
Order details survive a POS reload, and you can re-open the modal ("Change") to edit them later.

5. In the cart: options, quantity, payment

  • Tap a laundry line to open its options (variants/attributes). Re-opening shows the previous choices; you can change them anytime.
  • Quantity / weight is set with the numpad. Wash-Dry-Fold is editable (per KG); Dry/Wet Clean and Shoe Clean stay at 1.
  • Lines are shown in a fixed order: Wash-Dry-Fold, Press, Dry/Wet Clean, Shoe Clean, then anything else.
  • Payment is blocked if any laundry line still has unselected options, or if a Wash-Dry-Fold line is under the minimum weight (6KG for a single line, 4KG per line when there are several).

6. The receipt

A laundry order prints multiple copies, each cut separately by the thermal printer:

  • one TRANSACTION COPY per service line (numbered "1/n", with that line boxed);
  • a SHOP COPY;
  • a CUSTOMER COPY for Drop-off, Drop-off & Delivery and Self-service.

Each receipt shows the customer, tags, service type, turnaround and the pickup/claim/delivery schedule, plus write-in count lines for Wash-Dry-Fold and Press.

7. Opening & closing the register

To enforce cash accountability, the cashier must count the drawer by denomination (the manual amount box is read-only):

  • Opening — the counted total must match the previous session's closing cash, and the cashier ticks an accountability checkbox, before Open Register is enabled.
  • Closing — the cash variance must be exactly zero and the checkbox ticked before Close Register is enabled. (Auto-fill is disabled.)
